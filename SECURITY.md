# Security Policy

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities via GitHub Issues.**

We use [HackerOne](https://hackerone.com) for responsible disclosure. To report a vulnerability:

1. Go to our HackerOne program and submit a report.
2. Include a clear description of the issue, steps to reproduce, and potential impact.
3. We will respond within **72 hours** and keep you informed of our progress.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |

## Scope

In scope:
- BasePulse application (API, frontend, smart contracts)
- Authentication and authorization flaws
- Injection vulnerabilities (SQL, XSS, etc.)
- Sensitive data exposure

Out of scope:
- Denial of service attacks
- Social engineering
- Third-party services
